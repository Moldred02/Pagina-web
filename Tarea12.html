Introducción



Ha sido utilizado principalmente, en el desarrollo de software de control, de tiempo real y de misión crítica. Típicamente, estos sistemas son responsables de procesos industriales y militares, muy costosos, y en los cuales incluso vidas humanas dependen del buen funcionamiento del software. Es vital en tales sistemas, utilizar un lenguaje que como Ada, ayuda en la creación de software de alta calidad.

Símbolos



  • => es usado en when,cases
  • .. es usado para rangos
  • ** es para exponenciación
  • /= es No igual
  • >= es mayor o igual
  • <= es menor o igual
  • <> es Arreglos


Sintaxis



  • Es indiferente el uso de mayúsculas y minúsculas en los identificadores y palabras claves

  • Todo el programa es un único procedimiento, que puede contener subprogramas

  • Cada sentencia se cierra con end.

  • El operador de asignación es :=, el de igualdad =.

  • Subprogramas
    • Definen algoritmos y se clasifican en procedimientos (procedure) y funciones (function).

  • Package
    • Definen módulos que son colecciones de entidades.

  • Tareas
    • Definen acciones que pueden ejecutarse en paralelo con otras.

  • Unidades protegidas
    • Sirven para coordinar la compartición de datos entre tareas concurrentes.

  • Unidades genéricas
    • Sirven para definir componentes reusables.

Ejemplo





 package FUERZA-NAVA is
--
	type BUQUE-DE-GUERRA is	(CRUCERO, FRAGATA, SUBMARINO);	
	subtype	CANTIDAD is	INTEGER	range	Ø..1.ØØ;
	type TIPO-DE-BUQUE is
	record	:	
		NOMBRE: BUQUE-DE-GUERRA;
		NUMERO-DE-BUQUES. CANTIDAD;
	end	record;	
--
	task NAVES-ACTIVAS	is
	entry CAMBIOS (NAVES-ACTOVAS: out TIPO-DE-BUQUE);
	end	NAVES-ACTIVAS;	
end	FUERZA-NAVAL:
